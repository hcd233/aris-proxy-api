# Dataset 页面 stepper 重设计

## Status

accepted

## Context

`web/src/app/(dashboard)/dataset/page.tsx` 当前布局为"左筛选栏 + 右主区"双栏结构，存在以下问题：

1. **信息重复** — 模型分布同时出现在左侧统计卡和右侧"Dataset Summary"卡，同一数据展示两遍。
2. **流程断裂** — 用户心智模型是 `筛选 → 预览 → 导出`，但布局将统计信息拆分在左右两栏，无连贯叙事。
3. **导出 CTA 弱** — 导出按钮是底部一个浮动 div，视觉上不像是最终动作。
4. **层级缺失** — 筛选、统计、预览、导出视觉权重相同，互相竞争注意力。

## Decision

将页面重设计为 **三步向导（stepper）纵向流**，所有步骤同时可见，顶部 stepper 指示器标记进度：

1. **Step 1 — Configure**：筛选条件（分数滑块、模型选择、时间范围）
2. **Step 2 — Review**：统计概览（总数 + 分数分布 + 模型分布，去重）+ 可折叠格式预览
3. **Step 3 — Export**：导出确认面板（筛选摘要 + 总数 + 格式说明 + 大导出按钮）

### 关键设计决策

- **自由跳转**：步骤间不锁定，stepper 是视觉叙事设备而非导航控制器。所有步骤同时渲染，用户可自由滚动。改筛选条件时统计原地刷新，不跳转。
- **格式预览可折叠**：默认折叠，点击展开查看 ShareGPT JSON。不占据主视觉空间（用户确认其为"辅助确认"用途）。
- **分数分布保持 badge**：不做成图表，仅消除重复（从两张卡合并为一张）。
- **导出确认面板**：步骤 3 展示筛选摘要（分数阈值、模型列表、时间范围、总数、格式），配合大导出按钮，强化"最终动作"感。

## Considered Options

1. **单列纵向流（无 stepper）** — 简单但缺乏引导感，用户可能不知道流程顺序。否决。
2. **左筛选栏 + 右主区（现状修复）** — 保留双栏但去重。仍存在流程叙事断裂问题。否决。
3. **严格锁定向导** — 必须完成步骤 1 才能解锁步骤 2。对数据探索工具过于限制。否决。
4. **三步自由跳转向导（选定）** — stepper 提供视觉引导 + 自由跳转 + 原地刷新，兼顾引导性和灵活性。

## Consequences

- 页面从双栏改为单列纵向流，stepper 指示器在顶部。
- 模型分布和分数分布各只出现一次（在 Step 2），消除重复。
- 导出按钮从浮动 div 升级为确认面板中的主 CTA。
- 格式预览默认折叠，减少视觉噪音。
- 新增 i18n key：`dataset.step_*`、`dataset.export_summary`、`dataset.export_format` 等。
- stepper 为页面内联实现，不提取为通用组件（YAGNI，仅此页使用）。
