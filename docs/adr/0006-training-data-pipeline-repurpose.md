# 治理功能重新定位为训练数据管线

## Status

accepted

## Context

项目已有的治理功能（SessionScore 评分、SessionDedup 去重、ThinkExtract 思维链提取、Blocked 内容屏蔽）在自用场景下存在"有数据没闭环"的问题：cron 持续生成评分、摘要、思维链数据，但这些数据没有下游消费链路——评分不影响排序/过滤，摘要不进搜索，思维链提取后无人看。

grill 讨论中确认产品定位为"自用增强"，下一步方向是"将沉淀的对话数据用于训练私有小模型（SFT 风格克隆）"。这意味着治理功能不是"治理"，而是**训练数据清洗管线**：评分 = 样本质量标注，去重 = 训练样本去重，思维链提取 = 推理过程数据提取，内容屏蔽 = 脏数据过滤。

## Decision

将治理功能的语义重新定位为训练数据管线。下一步迭代建设 DatasetExport（数据集导出）能力：按评分阈值筛选会话 → 统计预览 → 流式导出 ShareGPT JSONL。治理功能产出的数据作为管线的输入，不再追求独立的"治理闭环"（如评分驱动排序、摘要驱动搜索）。

## Considered Options

1. **继续做治理闭环**（评分驱动排序/过滤/搜索）— 数据消费场景是会话检索与导航，ROI 不如训练数据导出直接。
2. **砍掉治理功能**（评分/去重/思维链提取是过度设计）— 基础设施已建好，砍掉浪费已有投入，且这些功能恰好是训练数据清洗需要的。
3. **重新定位为训练数据管线**（选定）— 复用已有基础设施，改变消费终点，ROI 最高。

## Consequences

- SessionScore 从"会话质量评分"变为"SFT 样本质量标注"，语义扩展但不冲突。
- ThinkExtract 从"展示用思维链提取"变为"SFT 推理数据提取"，已有 cron 逻辑无需改动。
- SessionDedup 和 Blocked 的语义不变，它们本身就是数据清洗。
- 后续不再投入"评分驱动排序"等治理闭环功能，除非训练数据管线之外有新的独立需求。
